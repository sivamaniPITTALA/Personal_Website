9. Is your webpage responsive? What do you mean by it?

✅ Yes, my webpage is responsive.

📱 What is Responsive Design?

Responsive web design means that the website automatically adjusts its layout and content to look good on all devices, from desktop computers to tablets and mobile phones. This is achieved using:

  • 📐 Flexible grid layouts using percentages
  • 📱 CSS media queries to detect screen size
  • 🖼️ Fluid images that scale appropriately
  • 🔄 Adjustable navigation for smaller screens

💻 Media Query Example:

@media (max-width: 768px) { nav ul { flex-direction: column; } }

Before: Users had to manually scroll through all 6 hobby cards to find what they wanted.

After: Added a search input that filters cards in real-time as the user types. If a user types "reading", only the "Reading Technical Books" card remains visible. This saves time and improves user experience significantly.

6. Which mouse and keypress elements did you use?

🖱️ Mouse events used:

  • mouseenter - Triggered when mouse enters a card, adds highlight effect
  • mouseleave - Triggered when mouse leaves a card, removes highlight
  • click - Used for scroll-to-top button, form submission, and navigation
  • scroll - Detects page scroll position to show/hide scroll-to-top button

⌨️ Keypress events used:

  • input - Used in search filter to detect keystrokes and update results in real-time
  • change - Used in form inputs and dropdowns

7. How does mouseover event differ in its effects from mouseenter and mouseleave events?

🔄 Event Differences:

mouseover:

  • Fires when the pointer enters an element OR any of its descendants
  • Bubbles up through the DOM hierarchy
  • Can trigger multiple times when moving between child elements
  • Use case: When you need to detect entry into a container and all its children

mouseenter:

  • Fires only when the pointer enters the element itself (not descendants)
  • Does NOT bubble - stays on the target element
  • Triggers only once when entering the element
  • Use case: Highlighting a card when the mouse enters its main area

mouseleave:

  • Fires when the pointer leaves the element itself (not descendants)
  • Does NOT bubble - stays on the target element
  • Triggers only once when leaving the element
  • Use case: Removing highlight when mouse leaves the card area

💡 Practical Example:

In my website, I use mouseenter and mouseleave for card effects because I want the highlight to apply only when entering the card's main area, not when moving between child elements inside the card (like links or text).
