7. How does mouseover event differ in its effects from mouseenter and mouseleave events?

ğŸ”„ Event Differences:

mouseover:

  • Fires when the pointer enters an element OR any of its descendants
  • Bubbles up through the DOM hierarchy
  • Can trigger multiple times when moving between child elements
  • Use case: When you need to detect entry into a container and all its children

mouseenter:

  • Fires only when the pointer enters the element itself (not descendants)
  • Does NOT bubble - stays on the target element
  • Triggers only once when entering the element
  • Use case: Highlighting a card when the mouse enters its main area

mouseleave:

  • Fires when the pointer leaves the element itself (not descendants)
  • Does NOT bubble - stays on the target element
  • Triggers only once when leaving the element
  • Use case: Removing highlight when mouse leaves the card area

ğŸ’¡ Practical Example:

In my website, I use mouseenter and mouseleave for card effects because I want the highlight to apply only when entering the card's main area, not when moving between child elements inside the card (like links or text).
